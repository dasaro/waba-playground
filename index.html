WABA Syntax Guide

ASP (Answer Set Programming) Basics

  • Comments: % This is a comment
  • Facts: fact_name.
  • Rules: head :- body1, body2.
  • Constraints: :- constraint_violated.
  • Choice: { chosen_atom }.

WABA Core Predicates

Assumptions (Defeasible Atoms)

assumption(atom_name).

Declares a defeasible assumption that can be in or out of extensions.

Weights

weight(atom_name, weight_value).

Assigns a positive integer weight to an atom (default: 1).

Rules (Inference)

head(rule_id, conclusion).
body(rule_id, premise1).
body(rule_id, premise2).

Defines inference rules: conclusion ‚Üê premise1, premise2

Compact form: head(r1, conclusion; r1, premise1; r1, premise2).

Contraries (Attack Relation)

contrary(attacked_assumption, attacking_element).

Important: "attacking_element attacks attacked_assumption" (not the reverse).

Budget Constraint

budget(beta).

Sets the budget Œ≤ for attack resolution costs.

Example Framework

% Assumptions
assumption(assume_a).
assumption(assume_b).

% Weights
weight(assume_a, 10).
weight(assume_b, 20).
weight(attack_element, 15).

% Rules
head(r1, attack_element).
body(r1, assume_a).

% Contraries
contrary(assume_b, attack_element).

% Budget
budget(50).

Special Constants

  • #sup - Superior (infinity-like value)
  • #inf - Inferior (negative infinity-like value)
